## Codebase Patterns
- Module path is `github.com/your-org/dashboard`
- Config loaded from `config.yaml` (YAML) via `gopkg.in/yaml.v3`
- Env vars loaded from `.env` via manual key=value parse in `internal/config/env.go` (no third-party library)
- Placeholder package files (single `package <name>` line) are used to satisfy directory requirements without real implementations
- `go vet ./...` is the typecheck command for this project
- `go build ./...` for full build verification
- Migration files use 4-digit prefix format: `0001_name.up.sql` / `0001_name.down.sql`
- Migrations embedded via `//go:embed migrations` in `internal/store/embed.go`; variable `migrationsFS embed.FS`
- golang-migrate wired with `database/sqlite` (modernc.org/sqlite, pure Go) + `source/iofs`; driver name is `"sqlite"` (sql.Open)
- nullable DB columns → `sql.NullString`, `sql.NullInt64`, `sql.NullTime` in model structs
- `Store.db` is unexported `*sql.DB`; use `store.New(path)` to open + migrate in one step
- Store query methods: use `ExecContext` + `LastInsertId` for INSERT (not RETURNING), then fetch by ID
- Test helper: use `t.TempDir() + "/test.db"` with `New()` for store tests (avoids `:memory:` connection-sharing issues with golang-migrate)
- Standard testing package (no testify); use `t.Fatal`/`t.Error` directly
- `scanMember` takes `*sql.Rows` (not `*sql.Row`) since team_members uses QueryContext for multi-row scans
- **TIME COMPARISON GOTCHA**: modernc/sqlite driver stores Go `time.Time` params in RFC 3339 format; SQLite's `CURRENT_TIMESTAMP` uses `"YYYY-MM-DD HH:MM:SS"`. These formats do NOT compare correctly with `<`/`>`. Fix: format Go time params explicitly with `.UTC().Format("2006-01-02 15:04:05")` when they must compare against `CURRENT_TIMESTAMP`
- **AI cache prune**: use `cutoff := time.Now().UTC().Add(-olderThan).Format("2006-01-02 15:04:05")` then `WHERE created_at <= ?` (note `<=` to include same-second entries when olderThan=0)
- **NULL-safe upsert for ai_cache**: SQLite UNIQUE with NULL team_id doesn't conflict; use SELECT-then-INSERT/UPDATE pattern (same as source_configs)
